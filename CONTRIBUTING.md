# Contributing to ASPS

Thank you for your interest in **AI Software Production Systems (ASPS)**. This is an open research framework, and it gets stronger through critique, refinement, and real-world testing. Contributions of all kinds are welcome.

## Ways to contribute

ASPS is a *research framework* (a thesis + reference architecture), not a software library — so most contributions are intellectual rather than code.

| Contribution | How |
|--------------|-----|
| 🐛 **Corrections** | Found a typo, broken link, or factual error? Open an [Issue](../../issues) using the *Correction* template. |
| 💡 **Framework critique** | Disagree with a claim, see a gap, or have a counter-example? Open an [Issue](../../issues) using the *Framework Feedback* template, or start a [Discussion](../../discussions). |
| 📊 **Case studies** | Applied ASPS (or part of it) in a real system? Share what held and what broke — this is the most valuable contribution of all. |
| ✍️ **Clarity & docs** | Improvements to the README, diagrams, or explanations are welcome via Pull Request. |
| 🔗 **References** | Suggest relevant prior art, standards, or related work that should be cited. |

## Guidelines

1. **Be specific.** "Section 14 contradicts Section 19 because X" is far more useful than "this is wrong."
2. **Be constructive.** The goal is a framework the industry can rely on — critique the idea, not the person. See our [Code of Conduct](CODE_OF_CONDUCT.md).
3. **Cite sources** where relevant, especially for counter-claims or prior art.
4. **Scope.** ASPS is intentionally vendor-neutral and conceptual. Proposals should preserve that — control responsibilities, not specific products.

## Pull requests

For text/diagram/documentation changes:

1. Fork the repository and create a branch (`fix/typo-section-14`).
2. Make your change.
3. Open a PR with a clear description of *what* changed and *why*.

Substantive changes to the framework's claims are best discussed in an Issue **first**, so we can align before you invest effort.

## Citing ASPS

If you build on ASPS, please cite it — see [`CITATION.cff`](CITATION.cff) or the citation block in the [README](README.md).
