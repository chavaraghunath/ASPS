# Security Policy

## Scope

This repository contains a **research framework and thesis document** (PDF, README, diagrams) — it does **not** ship executable code, dependencies, or a running service. As such, traditional software vulnerabilities (CVEs, dependency exploits) do not apply here.

However, we take the **integrity of the published research** seriously.

## What to report

Please report any of the following:

- **Integrity issues** — signs the PDF, DOI, or repository contents have been tampered with or misrepresented.
- **Attribution / provenance concerns** — suspected plagiarism, mis-citation, or unauthorized redistribution that violates the [CC BY 4.0 license](LICENSE).
- **Account or supply-chain concerns** — anything suggesting the repository or its linked records (Zenodo, etc.) have been compromised.

For **factual errors or framework critique**, please use a regular [Issue](../../issues) instead — that is not a security matter.

## How to report

Email the maintainer directly: **raghunath.chava@gmail.com**

Please include:
- A clear description of the concern
- Links or references (DOI, file, commit, URL)
- Any supporting evidence

You can expect an acknowledgement within **5 business days**.

## Verifying authenticity

The canonical, authoritative version of this work is the archived record:

- **DOI (concept):** [10.5281/zenodo.20549569](https://doi.org/10.5281/zenodo.20549569)
- **License:** CC BY 4.0
- **Author:** Raghunath Chava

If a copy disagrees with the Zenodo record, treat the **Zenodo record as authoritative**.
