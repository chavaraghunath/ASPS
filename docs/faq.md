# FAQ

Common questions about ASPS, answered concisely. For depth, follow the links into the wiki.

---

### What is ASPS, in one line?
A trustworthy, reversible, governed architecture for **autonomous enterprise software production** — optimizing *production correctness per unit cost and time*, not raw code output. See **[Core Concepts](core-concepts.md)**.

### Is ASPS a tool, a product, or a methodology?
None of those. It is a **vendor-neutral reference architecture and research framework** — defined by *control responsibilities*, implementable on any cloud, model provider, or toolchain. See **[Reference Architecture](reference-architecture.md)**.

### How is ASPS different from DevOps / MLOps / Platform Engineering?
Those each optimize a *slice* of delivery for human-paced change. ASPS governs the **whole intent-to-runtime chain** for **autonomous actors** acting at machine speed and consequence. Full comparison: **[Differentiation from Existing Paradigms](differentiation.md)**.

### What does CTQOST-R stand for?
**C**ost, **T**ime, **Q**uality, **O**perability, **S**ecurity, **T**rust, **R**eversibility — the seven-dimension objective function of ASPS. See **[CTQOST-R Optimization Model](ctqost-r.md)**.

### Why is reversibility singled out?
Because autonomous actions happen at machine speed and scale, an undefined recovery path becomes an uncontrolled incident. ASPS makes recovery a **promotion gate**: no reversal/compensation/containment/safe-degradation path, no release. See **[Reversibility by Design](reversibility-by-design.md)**.

### Do I need all eleven domains to start?
No. ASPS is adopted **incrementally** up the **[Maturity Model](maturity-model.md)**. Start with the control that closes your most dangerous gap — usually agent governance and reversibility. See the **[Adoption Roadmap](adoption-roadmap.md)**.

### Does ASPS replace my existing governance (COBIT, ITIL, NIST SSDF)?
No — it **aligns with** them. ASPS slots into your existing governance estate; NIST SSDF informs Cyber Production Security, COBIT/ITIL inform operating-model integration. See **[Reference Architecture](reference-architecture.md)** (Standards alignment).

### Is ASPS only for large enterprises?
It is designed for **regulated, high-scale, or mission-critical** production, where a weak domain lowers whole-system trust. Smaller teams can still adopt its principles (reversibility, evidence-led promotion) selectively.

### Where do I report a flaw or disagree with a claim?
Open a **[Framework Feedback issue](https://github.com/chavaraghunath/ASPS/issues)** or a **[Discussion](https://github.com/chavaraghunath/ASPS/discussions)**. Critique strengthens the framework — see the repo's `CONTRIBUTING.md`.
